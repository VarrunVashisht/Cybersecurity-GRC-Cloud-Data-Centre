# Data Classification Framework

## 1. Purpose
To classify data based on sensitivity, regulatory impact, and business criticality.

## 2. Data Classification Levels

### Public
- Marketing content
- Public documentation
- Website content

### Internal
- Internal policies and procedures
- Employee operational documents
- Non-sensitive system documentation

### Confidential
- Customer business data
- Internal financial reports
- System architecture diagrams

### Restricted
- Customer personally identifiable information (PII)
- Financial records
- Healthcare data (PHI)
- Authentication credentials and encryption keys

## 3. Handling Requirements
- Public: No restriction
- Internal: Authorized employees only
- Confidential: Role-based access, encryption
- Restricted: Strong access control, encryption, logging, and monitoring
